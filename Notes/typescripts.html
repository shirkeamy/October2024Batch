TypeScript Assigment (Theory) Solution

  1. TypeScript is a strongly typed superset of JavaScript that compiles to plain JavaScript.
  2. It adds optional static typing, interfaces, and advanced features for better maintainability.
  3. TypeScript is known as an Object-oriented programming language whereas JavaScript is a prototype-based language
Differences between JavaScript and TypeScript
TypeScript JavaScript
TypeScript can be strongly typed JavaScript is dynamically typed only
TypeScript is more readable and maintainable JavaScript not much readable and maintainable
TypeScript supports abstraction through interfaces JavaScript does not
TypeScript code is easier to debug as the codebase expands, because type errors can be discovered at compilation time JavaScript does not detect errors at compiletime

TypeScript helps in catching errors at compile-time, improving code quality and readability. It also supports modern JavaScript features and object-oriented programming concepts.

It enforces type safety, reducing bugs caused by unexpected data types.

TypeScript makes the codebase more readable and maintainable

TypeScript offers intellisense, making development faster and reducing the chances of errors.

TypeScript includes tuples, enums, generics, and type interfaces, which JavaScript lacks

Since TypeScript checks types before runtime, it prevents many common runtime errors.
Example: If a function expects a number but gets a string, TypeScript will flag an error before execution.

  • Install Node.js and npm from the official website.
  • Install TypeScript globally using npm install -g typescript.
  • Install VS Code as an editor.
  • Verify installation using tsc --version.

Windows restricts script execution for security reasons.

It can be resolved using the command Set-ExecutionPolicy RemoteSigned -Scope CurrentUser in PowerShell.
