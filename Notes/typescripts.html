  2. It adds optional static typing, interfaces, and advanced features for better maintainability.
  3. TypeScript is known as an Object-oriented programming language whereas JavaScript is a prototype-based language

Both null and undefined represent absence of a value, but they behave differently in TypeScript.

undefined

Represents a variable that has been declared but not assigned a value

used as the default return value of functions that don't explicitly return anything

Example 
let x: undefined;
console.log(x); // Output: undefined

let y;
console.log(y); // Output: undefined (Implicitly assigned)

function doNothing(): void {
    // No return statement
}
console.log(doNothing()); // Output: undefined

null

Represents an explicit absence of value

It’s a deliberate assignment to indicate "no value"

null must be manually assigned 

let person: string | null = null; // Explicitly assigned
console.log(person); // Output: null

function getUser(): string | null {
    return null; // Intentionally returning null
}

An interface in TypeScript defines a contract for an object, specifying the properties and methods it must have. It helps with type checking and code organization

Interfaces only exist at compile time and do not generate JavaScript code

Used for type safety and reusability

Can be implemented by classes and extended by other interfaces

Help in enforcing consistency across different parts of an application 

interface Person {
    name: string;
    age: number;
}

const user: Person = {
    name: "Alice",
    age: 25
};

console.log(user.name); // Output: Alice

An Enum (Enumeration) in TypeScript is a special type that assigns names to a set of numeric or string values.

It helps organize related constants, making the code more readable and maintainable

Enums are mainly used when a variable can take a fixed set of values, such as days of the week, user roles, or status codes.

enum's are efficient and automatically indexed

Example 
enum Role {
    Admin,
    User,
    Guest
}

let userRole: Role = Role.Admin;
console.log(userRole); // Output: 0
console.log(Role.User); // Output: 1
                                

enum DaysOfWeek {
    Sunday = 0,
    Monday = 1,
    Tuesday = 2,
    Wednesday = 3,
    Thursday = 4,
    Friday = 5,
    Saturday = 6
}

console.log(DaysOfWeek.Monday); // Output: 1
console.log(DaysOfWeek[3]); // Output: "Wednesday" (Reverse Mapping)

The if-else statement is used for conditional execution

It allows you to execute different code blocks based on whether a condition is true or false.

Example 1 - Check positive or negative number 
let num: number = 10;

if (num > 0) {
  console.log("The number is positive.");
} else if (num < 0) {
  console.log("The number is negative.");
} else {
  console.log("The number is zero.");
}
Example 2 - Check Even or Odd number 
function checkEvenOdd(num: number): string {
    if (num % 2 === 0) {
        return "Even number";
    } else {
        return "Odd number";
    }
}

console.log(checkEvenOdd(7)); // Output: Odd number
console.log(checkEvenOdd(10)); // Output: Even number

The switch-case statement is used when you need to compare a single value against multiple possible options

It is more efficient and readable than multiple if-else statements when dealing with many conditions.

Each case is checked against the expression

The break statement prevents fall-through (without break, execution continues to the next case)

The default case executes if no match is found

Example 1 - Days of the Week 
let day: number = 3;

switch (day) {
  case 1:
    console.log("Monday");
    break;
  case 2:
    console.log("Tuesday");
    break;
  case 3:
    console.log("Wednesday");
    break;
  case 4:
    console.log("Thursday");
    break;
  case 5:
    console.log("Friday");
    break;
  case 6:
    console.log("Saturday");
    break;
  case 7:
    console.log("Sunday");
    break;
  default:
    console.log("Invalid day number");
}
Example 2 - User Roles 
let role: string = "admin";

switch (role) {
  case "admin":
    console.log("You have full access.");
    break;
  case "editor":
    console.log("You can edit content.");
    break;
  case "viewer":
    console.log("You can view content.");
    break;
  default:
    console.log("Invalid role.");
}

A function in TypeScript is a reusable block of code that performs a specific task. It helps organize code, avoid repetition, and improve maintainability.

Functions can accept parameters, return values, and have type annotations to ensure type safety

  1. Regular/Normal Function / Named Function
  2. Variable as function / Anonymous Function
  3. Arrow Function / Lambda Function
  4. Function with Default Parameters
  5. Function with Optional Parameters

Exmaple 1 - Regular/Normal Function / Named Function 

function greet(name: string): string {
    return `Hello, ${name}!`;
}

console.log(greet("Alice")); // Output: Hello, Alice!
Exmaple 2 - Variable as function / Anonymous Function 
const add = function (a: number, b: number): number {
    return a + b;
};

console.log(add(5, 3)); // Output: 8
Exmaple 3 - Arrow Function / Lambda Function

Type 1 

const multiply = (a: number, b: number): number => a * b;

console.log(multiply(4, 2)); // Output: 8

Type 2 

const multiply = (a: number, b: number): number => {
    return a * b;
} 

console.log(multiply(4, 2)); // Output: 8

Type 3 

const multiply: (a: number, b: number) => number
= (a: number, b: number) => {
        return a * b;
}

console.log(multiply(4, 2)); // Output: 8
Exmaple 4 - Function with Default Parameters 
function greetUser(name: string = "Guest"): string {
    return `Hello, ${name}!`;
}

console.log(greetUser()); // Output: Hello, Guest!
Exmaple 5 - Function with Optional Parameters 
function displayAge(name: string, age?: number): string {
    return age ? `${name} is ${age} years old.` : `${name}'s age is unknown.`;
}

console.log(displayAge("Alice")); // Output: Alice's age is unknown.
console.log(displayAge("Bob", 25)); // Output: Bob is 25 years old.

Arrow functions (=>) are a shorter syntax for writing functions in TypeScript. They are also called lambda functions and are primarily used for concise one-liners, callbacks, and functional programming.

  • If the function has only one statement, {} and return can be omitted.
  • If multiple statements exist, {} and return must be used.
